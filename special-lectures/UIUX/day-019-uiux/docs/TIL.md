## 과거의 레이아웃

- 웹 초반에 position, float 등을 이용해 디자인
- **요즘에는 flex, grid를 사용**

### position

#### static

- top, bottom 등 적용 안됨

#### relative

- 자기 자신을 기준으로 이동
- **원래 자리가 그대로 유지됨** : 기존 자리를 유지하며 전체적인 레이아웃을 깨뜨리지 않으면서 사용(실무에선 거의 사용 x)

#### absolute

- 가장 가까운 **(static 아닌) 부모**를 기준으로 이동
- 위치를 자유롭게 지정 가능
- reative와 반대로 **원래 위치를 완전히 버림** : 다음 요소가 absolute의 자리로 이동(겹쳐질 수 있음)

> [TIP] **실무에서는 부모 : `relative`, 자식 : `absolute` 형태로 사용**
> `relative`는 부모에 주는 용도 외에 잘 안씀

#### fixed

- 부모와 상관 없이 화면 기준(브라우져)으로 고정
- 스크롤과 상관 없이 고정

#### float

- 자연스러운 텍스트 흐름을 위한 배치
- 레이아웃 도구가 아님
- `clear`: float의 영향을 받지 않으며 바로 밑으로 내려가는 설정(block 의 여백도 없음)

> [!Warnning]
> 부모가 자식의 높이를 인식 못함
> 요소가 예상과 다르게 배치
